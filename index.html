Git Uitleg

In deze tutorial wil ik je graag uitleggen hoe je met Git kan omgaan met behulp van de GIT GUI.

In simpele termen is Git een versiebeheersysteem. Dit betekent dat het een programma is dat wordt gebruikt om wijzigingen in bestanden bij te houden en te beheren. Dit is vooral handig als je aan een project werkt met meerdere mensen, omdat het je toelaat om gemakkelijk te zien wie welke wijzigingen heeft aangebracht en om conflicterende wijzigingen op te lossen.

Om Git te gebruiken, moet je eerst een repository (een map waarin je projectbestanden worden opgeslagen) aanmaken op je computer. Vervolgens kun je bestanden toevoegen aan de repository en wijzigingen aanbrengen in die bestanden. Elk moment dat je wijzigingen aanbrengt, kun je deze "committen" naar de repository, wat betekent dat je de wijzigingen opslaat en er een versiehgeschiedenis van maakt. Dit maakt het gemakkelijk om terug te gaan naar een eerdere versie van een bestand als dat nodig is.

Maak een map in Windows voor je repository (opslag).

Ga naar het mapje, klik met je rechter muisknop in de map, klik op "GIT GUI" en daarna "Create New Repository". Geef het een leuke naam en klik op "Create".

Goed gedaan! Je hebt nu een repository aangemaakt. Nu kan je beginnen met het toevoegen van een bestand aan je repository, met als voorbeeld een tekst document.

Volgende

Maak een tesktbestand aan met de naam "Leuke Tekst" in jouw aangemaakte repository.

Open het tesktbestand en schrijf er iets leuks in. Gebruik als voorbeeld "Mijn eerste zin". Vergeet niet om het op te slaan!

Ga nu terug naar het GIT GUI programma en klik op Rescan. Nu zie je jouw bestand en tekst maar pas op, het is nog niet opgeslagen in de repository!

Volgende

Om je tesktbestand op te slaan in de repository moeten we het eerst Stagen.
Klik op Stage Changed in GIT GUI en klik ja op de prompt

Nu is onze wijziging opgeslagen maar niet gecommit, dit moeten we doen zodat Git weet welke wijzigingen in welke bestanden wij willen opslaan (committen)

Voordat we kunnen committen moeten we een notitie achterlaten over onze commit, dit doen we door in het lege vak naast Rescan / Stage Changed te schrijven. Schrijf bijvoorbeeld "Ik heb een tekstbestand aangemaakt" en klik op Commit.

Goed gedaan! Je hebt nu je eerste commit opgeslagen!

Volgende

Ga nu terug naar je tekstbestand en schrijf "Mijn tweede zin" op de tweede regel en sla het op.

In GIT GUI klik op Rescan. Nu zie je jouw tweede zin samen met de eerste zin. Klik op Stage Changed & daarna op Commit
Gebruik als commit notitie "Mijn tweede zin" en klik op Commit.

Nu hebben we twee commits opgeslagen in onze repository. We kunnen nu ons bestand terugzetten naar bijvoorbeeld de eerste commit.

Volgende

In GIT GUI klik op Repository en daarna op Visualize master's History, nu zie je aan de linkerkant je 2 commits staan.

Klik op je eerste commit en kopieer de SHA1 ID, het is een regel met willekeurige letters & cijfers. Zoals dit: 06f8d067b12530c942e8049a433c07e122eb9cfd

Deze ID is specifiek aan deze commit, elke commit heeft een ander ID. Dus als je een andere commit wilt terugzetten moet je de ID van die commit gebruiken.

Ga terug naar het beginscherm van GIT GUI en klik bovenin op Branch & daarna op Checkout. Tevens kun je ook sneltoets CTRL+O gebruiken.

Volgende

Plak de ID van je eerste commit in het vak Revision Expression en klik op Checkout. Je ziet nu dat je bestand teruggezet is naar de eerste commit.

Als je jouw bestand nu opent zie je de tweede zin niet meer. Dit is erg handig als je de hele tijd fout bezig was en niet meer kan CTRL+Z'en. Nu gaan we beginnen met het maken van een Branch (tak) in onze repository.

Volgende

Momenteel hebben we maar 1 tak, dat is onze Master tak. We gaan nu een nieuwe tak maken, deze tak noemen we Test.

Dit zou je bijvoorbeeld gebruiken om dingen uit te testen zonder dat je de master branch aanpast.

Bovenin GIT GUI klik je op Branch en daarna op Create. Geef de nieuwe tak de naam Test en klik op Create.

Volgende

Nu hebben we 2 takken, de Master tak en de Test tak. Momenteel zit je in de Test tak, dit kun je zien linksonderin.
Maak nu een nieuwe zin in je tekstbestand, als voorbeeld "Test Regel" en sla het op. Ga terug naar GIT GUI en klik op Rescan.

Klik op Stage Changed. Gebruik als commit notitie "Test Regel" en klik op Commit.

Klik nu op Branch en daarna op Checkout, selecteer de Master tak en klik op Checkout.

Volgende

Ga nu terug naar je tekstbestand, je ziet dat de test regel er niet meer is. Dit komt omdat we in de Master tak zitten.

Dit maakt Git erg handig voor als je iets wilt uitproberen zonder de vrees dat je al jouw werk verpest.

Natuurlijk werkt dit voor de meeste bestandstypen en niet alleen voor tekstbestanden, maar Git werkt het best met tektsbestanden omdat je de verschillen ook echt kunt zien.
